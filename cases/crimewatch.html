CrimeWatch Pakistan: FIR Management

A full-stack crime reporting system featuring a 3NF PostgreSQL database, secure Django backend, and live analytical visualizations.

Full-Stack Database Architect
PostgreSQL, Django, SQL, Chart.js
Strict 3rd Normal Form (3NF)
FIR Filing, Status Tracking, Analytics

Overview

CrimeWatch Pakistan modernizes First Information Report (FIR) logging and case tracking. Law enforcement agencies often suffer from fragmented paper records or un-indexed databases, hindering cross-district investigation and spatial crime tracking.

Database Architecture & Security

The core of CrimeWatch Pakistan is a robust relational database schema designed to eliminate redundancy while preserving auditability:

  • Normalized Schema (3NF): Separated Complainants, Police Stations, Suspects, Officers, and Case Logs into distinct tables with foreign key constraints.
  • Parameterized Raw SQL Queries: Used parameterized queries in Django to ensure immunity against SQL injection vulnerabilities while achieving maximum query performance.
  • Dynamic Analytics Dashboard: Integrated Chart.js to render real-time graphs for crime frequency by category, station resolution rates, and monthly trends.
-- Relational Query Example: High-priority unresolved FIR breakdown SELECT ps.station_name, COUNT(f.fir_id) AS open_cases, f.crime_category FROM fir_records f JOIN police_stations ps ON f.station_id = ps.station_id WHERE f.status = 'UNDER_INVESTIGATION' AND f.severity_level >= 4 GROUP BY ps.station_name, f.crime_category ORDER BY open_cases DESC;

Key Outcomes

Delivered a fully operational management portal capable of handling high-volume incident logging with real-time analytics for police station administrators.
